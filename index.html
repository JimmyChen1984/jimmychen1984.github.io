移动芯片相关名词解释(后续更新)

发表于 2016-10-22   |   分类于 Mobile Chip   |     |   阅读次数

移动芯片的名词解释：
ARM =Advanced RISC Machines，全球领先的半导体知识产权(IP)提供商，全世界超过95%的智能手机和平板电脑都采用ARM架构，英国，现在被日本软银收购.
SoC芯片 =System-on-a-Chip,系统级芯片,也有称片上系统,是一个有专用目标的集成电路,其中包含完整系统并有嵌入软件的全部内容。同时它又是一种技术,用以实现从确定系统功能开始,到软/硬件划分,并完成设计的整个过程。

QC

QC =Qualcomm,美国高通公司，以在CDMA技术方面处于领先地位而闻名，是目前全球最大的手机CPU芯片厂商．
AP =Application Processor, 供操作系统、用户界面和应用程序运行，一般采用ARM芯片的CPU
BP =Baseband Processor, 供手机射频通讯控制软件运行，与AP分开
QSD =Quanlcomn Snapdragon, 高通骁龙，实际是msm，荣誉编号，很少用
APQ =Application Processor Qualcomn，高通应用处理器，实际就是没基带的soc。
QSC =Qualcomn Single Chip, 高通单芯片，功能机用的，处于弃用边缘
MDM =Mobile Date Modem, 移动数据基带，实际就是基带
MPQ =Media Processor Qualcomn,高通媒体处理器，智能电视，智能冰箱，无人驾驶汽车用的，大尺寸的APQ
MSM =Mobile Station Modem,apq+mdm,移动基带站，包含基带的SOC

MTK

MTK =MediaTek,台湾联发科技是全球著名IC设计厂商，专注于无线通讯及数字多媒体等技术领域

TI

TI =TEXAS INSTRUMENTS,德州仪器

国内

海思(Hisilicon),华为
展讯(Spreadtrum Communications),清华紫光
联芯(Leadcore),大唐电信

Android Watchdog 安卓系统服务看门狗实现原理

发表于 2016-09-23   |   分类于 Android frameworks   |     |   阅读次数

背景:　之所以我们的应用程序能够无忧无虑的畅快运行，正是源于SystemServer(SS)进程在后台默默无闻任劳任怨的提供各种系统核心服务, 例如　WindowManagerServer(WMS), ActivityManagerServive(AMS), PackageManagerServer(PMS)等等，这些服务一旦出现稳定性问题，整个应用程序根本就无法正常工作，对用户体验来说是灾难性打击. Android系统特别针对系统服务有可能出现不正常的情况引入Watchdog定时检查机制，一旦发现服务异常，大义灭亲地重启SS(用挥刀自宫会不会更恰当一些?),这样至少可以让SS在20s后又是一条好汉，继续服务于应用程序．
如果要分析Android系统中frameworks稳定性的问题，就需要我们要掌握Watchdog运行机制．本文结合Watchdog相关代码来介绍一下Watchdog的实现原理.
